Posted on 2019-10-05

网络协议

协议分层：下层为上层提供服务，同层必须相同。

分层独立的优势:独立功能，设计无需估计其他问题，只需要按照标准制作即可。
例如：使用 FTP 时，无需关心我走的线路是什么线。

ISO 国际标准组织提出OSI参考模型

PDU：Protocol Data Unit (协议数据单元)

端口号：客户端随机生成，服务端为固定。 每一个应用层软件都有一个随机端口号 0-65535，0-1000 特殊使用已经固定各个用途。

微软远程控制端口3389

TCP/IT封装

OSI 数据单元（PDU）
应用层
表示层
会话层
传输层 段(Fragment)
网络层 报文/IP包/包 (packet)
数据链路层 帧(Frame)
物理层 比特(bit)

TCP 有数据重传机制，而 UDP 没有。

应用层：HTTP:80 HTTPS:443  FTP:20/21   TFTP:69  Telnet:23  SMTP:25  POP3:110 DNS:53

传输层：TCP:6            UDP:17

网络层：ICMP:1           IP:0x0800         ARP:0x0806

数据链路层：MAC子层

TCP 序列号seq和确认号ack

确认号两个含义：
表示本序列号之前的顺序号，我都有确认收到
其次，表示下个包的序列号，应该+1
但在数据传输过程中，并不是一沉不变的+1，有时会加上数据长度。

IP与掩码计算

子网范围
255.0.0.0 / 8
255.255.0.0 / 16
255.255.255.0 / 24
255.255.255.255 / 32

步长：可以是IP范围的长度，也可以是网段范围的长度。
网段前缀：根据掩码来进行判断。

网络掩码只具有本地意义，只具有本地意义，只具有本地意义！ 如：A：192.168.1.1/16；B：192.168.1.129/25
192.168.1.1/16 —- ping —-> 192.169.1.129/25
无法ping通过。
因：ping包发出后，192.168.1.129/25 接收后，用自己的掩码进行判断，发现与自己不同，则丢弃。
注意：本机只看对方IP，并不看对方掩码。

子网掩码 > 24位（255.255.255.0）

10.11.100.1/26如何计算出起始IP与结束IP?
已知条件：
网段前缀：10.11.100
掩码：32 > 26 > 24，则使用32位.

计算方式：
IP步长：2^(子网范围 - 网段子网掩码) = 2^(32-26) = 2^6 = 64
每个子网段的起始地址为步长倍数：0、64、128、192
查看IP地址，主机位在0、64、128、192谁之间。发现在0-64之间。
网络ID：步长范围最左0-64,0在左。
广播ID：步长范围最右0-64,64在右。
起始IP：网络ID + 1
终点IP：广播ID - 1

即：

计算方式：
IP步长：2^(子网范围 - 网段子网掩码) = 2^(24-18) = 2^6 = 64

广播ID：步长范围最右64-128,128在右。
起始IP：网络ID + 1

网段起始IP，终点IP快速算法

算例

192.168.9.10 / 255.255.248.0的起始IP和结束IP是多少。

网段步长公式：256 - 非255字节 = 256-248 = 8
网段范围公式：终点网段 - 起始网段(0为起始) + 1 = 网段步长

即 起始IP：192.168.0.0
终点IP：192.168.15.255

全0和全1不能用

有效起始IP：192.168.0.1
有效终点IP：192.168.15.254

算例

192.168.14.10 / 255.255.252.0 / 22 的起始IP和结束IP是多少？
网段步长公式：256 - 非255字节 = 256 - 252 = 4
网段范围公式：终点网段 - 起始网段(0为起始) + 1 = 网段步长

有效起始IP：192.168.12.1
有效终点IP：192.168.15.254
