网络协议

协议分层：下层为上层提供服务，同层必须相同。

分层独立的优势:独立功能，设计无需估计其他问题，只需要按照标准制作即可。
例如：使用 FTP 时，无需关心我走的线路是什么线。

ISO 国际标准组织提出OSI参考模型

PDU：Protocol Data Unit (协议数据单元)

端口号：客户端随机生成，服务端为固定。 每一个应用层软件都有一个随机端口号 0-65535，0-1000 特殊使用已经固定各个用途。

微软远程控制端口3389

TCP 有数据重传机制，而 UDP 没有。

应用层：HTTP:80 HTTPS:443  FTP:20/21   TFTP:69  Telnet:23  SMTP:25  POP3:110 DNS:53

传输层：TCP:6        UDP:17

网络层：ICMP:1       IP:0x0800         ARP:0x0806

数据链路层：MAC子层

TCP 序列号seq和确认号ack

确认号两个含义：
表示本序列号之前的顺序号，我都有确认收到
其次，表示下个包的序列号，应该+1
但在数据传输过程中，并不是一沉不变的+1，有时会加上数据长度。

IP与掩码计算

10.11.100.1/26如何计算出起始IP与结束IP?(仅适用于/24位子网掩码)

已知条件：
掩码：最长32位
网段前缀：10.11.100

计算方式：
步长：2^(最长子网掩码 - 网段子网掩码) = 2^(32-26) = 64
每个子网段的起始地址为步长倍数：0、64、128、192
查看IP地址，主机位在0、64、128、192谁之间。发现在0-64之间。
网络ID：步长范围最左0-64,0在左。
广播ID：步长范围最右0-64,64在右。
